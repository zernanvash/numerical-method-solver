Modes of Operation

The console offers two main modes depending on the mathematical problem you want to solve. Toggle modes instantly using the MODE dropdown menu.

Root-Finding Mode

Solve equations of the form f(x) = 0. Ideal for tracing exact convergence values and plotting algebraic/transcendental functions.

  • Interactive parameter bounds
  • Live tangent and secant line plots
  • Tolerance matching down to 1e-15

ODE Simulation Mode

Solve initial value problems of the form dy/dt = f(t, y). Trace coordinates step-by-step and view vector-field trajectories.

  • Multi-variable function evaluations
  • Step size control (h)
  • Interactive step-by-step iteration reveal

Solver Methods

Numerical Method Solver implements five standard methods. Click on each method below to explore its description, formula, and requirements.

Bisection Method

Bracketing

A simple and robust root-finding method that repeatedly bisects an interval and selects a subinterval in which a root must lie.

Mathematical Formula
c = \frac{a + b}{2}

Where c is the midpoint. The algorithm checks the sign of f(a) * f(c) to determine the next interval.

Key Parameters Required

  • a: Lower boundary of the initial search interval.
  • b: Upper boundary of the initial search interval (must satisfy f(a) * f(b) < 0).
  • tolerance: Desired precision limit.
  • max iterations: Safety limit to prevent infinite loops.

Newton-Raphson Method

Open Method

An open method that uses the tangent slope of the function at the current guess to project forward and estimate the root.

Mathematical Formula
x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}

In this application, the derivative f'(x) is calculated dynamically using central finite differences.

Key Parameters Required

  • x0: The starting guess.
  • tolerance: Error threshold.
  • max iterations: Solver step limit.

Secant Method

Open Method

An open method that approximates the derivative using a secant line drawn between the two most recent approximations.

Mathematical Formula
x_{n+1} = x_n - f(x_n) \cdot \frac{x_n - x_{n-1}}{f(x_n) - f(x_{n-1})}

Requires two initial guesses but converges faster than Bisection without needing derivative analytical equations.

Key Parameters Required

  • x0: First initial guess.
  • x1: Second initial guess.
  • tolerance: Target error bound.

Euler's Method

First-Order ODE

A first-order numerical procedure for solving ordinary differential equations (ODEs) with a given initial value.

Mathematical Formula
y_{n+1} = y_n + h \cdot f(t_n, y_n)
t_{n+1} = t_n + h

Where h is the step size and f(t, y) represents the derivative dy/dt.

Key Parameters Required

  • t0: Initial value of the independent variable.
  • y0: Initial value of the dependent variable.
  • h: Step size.
  • steps: Total number of projection intervals.

Runge-Kutta 4th Order (RK4)

Fourth-Order ODE

A highly accurate fourth-order method that samples the slope of the differential equation four times across each step to estimate the trajectory.

Mathematical Formula
k_1 = f(t_n, y_n), \quad k_2 = f\left(t_n + \frac{h}{2}, y_n + h\frac{k_1}{2}\right)
k_3 = f\left(t_n + \frac{h}{2}, y_n + h\frac{k_2}{2}\right), \quad k_4 = f(t_n + h, y_n + h k_3)
y_{n+1} = y_n + \frac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4)

Key Parameters Required

  • t0, y0: Initial state coordinates.
  • h: Step size.
  • steps: Number of steps.

Math Expression Cheatsheet

The parser evaluates expressions typed into the FUNCTION panel. Use explicit multiplication (e.g. 2*x instead of 2x).

Category Syntax Description / Example
Operators +, -, *, /, ^ Standard arithmetic operations. Use ^ for exponentiation (e.g. x^2).
Trig Functions sin(x), cos(x), tan(x) Standard trigonometry. Trigonometric arguments evaluate in radians.
Logarithms log(x), ln(x), log10(x) ln(x) evaluates the natural logarithm, log10 is base-10.
Roots & Absolute sqrt(x), abs(x) Square root and absolute value computations.
Constants pi, e Mathematical constants. Case-insensitive (e.g. sin(pi) equals 0).
Variables x (Root)
t, y (ODE)		 Use x for root-finding. Use t (time) and y (value) for ODE.

Usage Workflow

Follow this standard operating procedure to compute solutions using the console application:

1

Select Problem Mode

Select Root-Finding Mode or ODE Simulation Mode in the top dropdown box. The inputs will automatically configure themselves for your selection.

2

Enter Equation & Parameters

Type your mathematical expression in the FUNCTION text field. Provide the parameters in the grid fields (initial guesses, step sizing, target tolerance, or max iteration bounds).

3

Calculate or Step Trace

Click [ SOLVE ] to execute the full calculations instantly. Alternatively, use [ STEP ] to run the solver row by row and watch the CRT graph animate the progress interactively.

4

Export Reports

Save your results for homework or research logs by clicking [ EXPORT TXT ] for a structured summary report, or [ EXPORT CSV ] to save raw numerical table data.
